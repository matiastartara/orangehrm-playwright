# 🎭 Storage Login — Playwright E2E Tests

E2E test automation project built with **Playwright** and **TypeScript**, implementing the **Page Object Model (POM)** pattern and **Storage State** to reuse the authenticated session across tests — avoiding repeated login flows on every run.

The application under test (AUT) is [OrangeHRM Live Demo](https://opensource-demo.orangehrmlive.com/).

---

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/storage-login-playwright.git
cd storage-login-playwright
```

### 2. Install dependencies

```bash
npm install
```

### 3. Install Playwright browsers

```bash
npx playwright install
```

> To install only Chromium for a minimal setup:
> ```bash
> npx playwright install chromium
> ```

---

## ▶️ Running the Tests

### Run all tests (setup + e2e)

```bash
npx playwright test
```

### Run only E2E tests (assumes `.auth/user.json` already exists)

```bash
npx playwright test --project=e2e-chromium
```

### Run only the authentication setup

```bash
npx playwright test --project=setup
```

### Run a specific test file

```bash
npx playwright test tests/e2e/userSearch.spec.ts
npx playwright test tests/e2e/directorySearch.spec.ts
```

### Run in UI mode (interactive visual runner)

```bash
npx playwright test --ui
```

### Run in headed mode (watch the browser)

```bash
npx playwright test --headed
```

## 🔐 Authentication Strategy (Storage State)

This project uses **Storage State** to persist the user session:

1. The `setup` project runs `tests/auth.setup.ts`, which performs the login on OrangeHRM and saves the session state (cookies and localStorage) to `.auth/user.json`.
2. The `e2e-chromium` project loads that state before each test, skipping the login step entirely.

> The `.auth/user.json` file is auto-generated and included in `.gitignore`.

## 🧩 Page Objects

### `BasePage`
Abstract base class extended by all Page Objects. Exposes reusable methods:
- `waitForPageLoad()` — waits for the `domcontentloaded` event
- `getTitle()` — returns the page title
- `reload()` — reloads the page and waits for it to load

### `AdminPage`
Page Object for the **Admin > User Management** section. Provides:
- `completeUsername(username)` — fills in the Username field
- `selectRole(role)` — selects a role from the dropdown
- `completeEmployeeName(name)` — fills and selects the employee name via autocomplete (waits for the API response)
- `setStatus(status)` — selects the user status
- `search()` — clicks the Search button
- `getRowByUsername(username)` — returns the row matching the given username
- `getCellFromRow(row, columnIndex)` — returns a specific cell from a row

### `DirectoryPage`
Page Object for the **Directory** section. Provides:
- `setJobTitle(title)` — selects a job title from the dropdown
- `setLocation(location)` — selects a location from the dropdown
- `search()` — clicks the Search button and waits for the API response
- `getCardResultSize()` — returns the total number of result cards
- `getCardResultAt(index)` — returns the card locator at a given index
- `getCardHeaderAt(index)` — returns the text of the card header at a given index

### `MenuPage`
Page Object for the **main navigation sidebar**. Centralizes sidebar navigation so tests don't hardcode `getByRole('link')` calls directly. Provides:
- `navigateToAdminPage()` — clicks the Admin sidebar link
- `navigateToDirectoryPage()` — clicks the Directory sidebar link
